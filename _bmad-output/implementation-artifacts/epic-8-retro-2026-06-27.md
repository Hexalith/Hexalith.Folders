## What Went Well

- The team converted assertions into executable evidence. REST parity, accessibility, baseline integrity, and C3 governance moved from planning prose or partial checks into gate-backed records.
- Story 8.3 corrected a high-risk test double problem. Rejection-flattening gateway stubs had been manufacturing false-green parity. The shared rejection-propagating gateway made canonical 409, safe denial, and gateway-hop ACL rejection behavior visible.
- The Story 8.5 to 8.6 split preserved governance integrity. The team did not fabricate Legal approval, did not hold completed engineering work hostage to an external gate, and later applied the C3 cascade only when recorded evidence existed.
- Reviews found real risks and improved evidence quality. Examples include stale counts in Story 8.3, missing tenant revocation coverage in Story 8.2, and exact Legal signer/date/location assertions in Story 8.6.
- The accessibility story used a union model instead of equating axe with all of WCAG. It combined axe AA tags, keyboard/focus checks, zoom/no-clipping checks, and existing bUnit evidence.

## Challenges And Growth Areas

- Several stories started from stale or incomplete premises. Story 8.1 found SDK/CLI/MCP wrappers did not imply server backing; Story 8.3 found idempotency conflict needed test propagation rather than production change; Story 8.5 found several residual-red categories were already green but still masked.
- Test evidence needed repeated hardening. Multiple review fixes corrected stale counts, missing negative paths, incomplete branch coverage, and report freshness.
- Some MVP read paths remain seed-backed or in-memory by default. Stories 8.1 and 8.2 correctly documented this as an MVP limitation, but production projection-backed implementations remain a follow-up concern.
- Safe-denial behavior required careful interpretation. The literal "ACL denial returns 403" expectation had to be reconciled with the verified layered-auth path that correctly returns 404 safe denial, while still proving 403 `folder_acl_denied` at the gateway-hop and adapter layers.
- Governance traceability has nuance. Story 8.6 showed that governance approval does not automatically mean every NFR reference-pending row must flip; C3 and C4 now both demonstrate approved governance with separately surfaced traceability gaps.

## Story Analysis Patterns

Common pattern: artifact claims were useful starting points, but direct source and gate verification changed the implementation plan in nearly every story.

Recurring review feedback:

- Reconcile story records with measured test counts before close-out.
- Prove negative paths, not only happy paths.
- Keep safe-denial and metadata-only behavior explicit in tests.
- Regenerate gate evidence at the reviewed commit.
- Make conformance flips lockstep with docs and scripts.

Breakthroughs:

- The shared rejection-propagating gateway became a reusable parity safety mechanism.
- The accessibility gate established a practical, non-brittle model for release-validation UI checks.
- The baseline re-mask guard in Story 8.5 turned "honest green" into an enforceable regression rule.
- The C3 cascade in Story 8.6 made governance YAML, docs, scripts, package gates, and conformance tests move together.

Technical debt and follow-ups:

- Story 8.1 left non-blocking route coverage gaps around some read safe-denial, 404, 503, and ACL list 200 cases.
- Story 8.1 documented the remaining `idempotency_key_not_accepted` vs `idempotency_key_not_allowed` inconsistency in pre-existing provider-readiness routes.
- Seed-backed diagnostics/read models should either become production projection-backed implementations or remain explicitly owned MVP limitations.
- Governance report freshness and exact approval evidence checks should be generalized beyond C3.

## Significant Discoveries

No product-direction replan is required from Epic 8. The discoveries were mostly evidence-integrity discoveries:

- SDK-generated coverage is not server-route coverage.
- Wire parity is not proven if the in-process gateway test double discards rejection identity.
- Axe is necessary but not sufficient for WCAG release evidence.
- A green test category can still be hidden by an obsolete CI filter.
- Governance approval and NFR traceability status are related but not identical contracts.

These do affect how later epics should be built and reviewed: evidence claims must be executable, current, and tied to the right layer.
