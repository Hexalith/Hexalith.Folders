# Story 7.14: Publish operations and audit documentation

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As an operator or audit reviewer,
I want operations-console and metadata-only audit documentation published,
so that production diagnosis and incident reconstruction are repeatable.

## Acceptance Criteria

> Epic 7.14 BDD from `_bmad-output/planning-artifacts/epics.md` (lines 1640–1651):
> Given operations console and audit surfaces exist
> When operations and audit documentation is published
> Then console workflows, metadata-only audit fields, redaction behavior, incident-mode use, alerting handoff, and backup/recovery expectations are documented
> And examples avoid file contents, provider tokens, credential material, secrets, and unauthorized resource details.

Decomposed acceptance criteria:

1. Publish the **operations-console operator-workflows** documentation (recommended `docs/operations/operations-console.md`). It must **summarize and cross-link**, not re-author, the Epic 6 UX sources (`docs/ux/ops-console-wireflows.md`, `docs/ux/ops-console-accessibility-and-no-mutation-verification.md`, `docs/ux/ops-console-performance-budget.md`). Document: the read-only console's **routed page inventory** — the **10 operator pages** Home `/`, Tenants `/tenants`, Folders `/folders`, FolderDetail `/folders/{folderId}`, Workspace `/folders/{folderId}/workspaces/{workspaceId}`, AuditTrail `/folders/{folderId}/audit-trail`, OperationTimeline `/folders/{folderId}/operation-timeline`, Provider `/folders/{folderId}/provider`, ProviderSupport `/providers/support`, IncidentStream `/_admin/incident-stream` (folder via `?folder=` **query**, not a path segment) — and explicitly call out the **2 dev-only galleries** RedactionGallery `/dev/redaction-gallery` and StateLabelGallery `/dev/state-label-gallery` as **non-production** (rendered only under `Env.IsDevelopment()`); the **3 critical operator journeys** (J1 Find-Workspace-and-Inspect-Trust-State, J2 Prove-Tenant-Isolation-and-Safe-Folder-Visibility, J3 Diagnose-Workspace-Failure-from-Evidence) with their route paths, terminal evidence surfaces, and the five trust questions (what happened / who-or-what caused it / when / from which surface / whether the evidence can be trusted); the host shape (`src/Hexalith.Folders.UI`, a **Blazor Web App host using Interactive Server rendering**, referencing **only `Hexalith.Folders.Client`** + `Hexalith.FrontComposer.Shell`, Dapr app id `folders-ui`, reading **only read-model projections** through the SDK — it never calls `AddHexalithEventStore` and there is no `/api/v1/commands` endpoint); and the **filter-rejection-only** reality (the console passes `filter: null`; the server rejects any non-null filter with `validation_error` / `filter_not_yet_supported`, todoRef `C4`). All examples metadata-only with opaque synthetic identifiers.

2. Document the **operator-disposition vocabulary as the primary visual** (F-4) inside the console doc. The vocabulary has **exactly 5 members** (wire form / English label): `auto_recovering` / "Auto-recovering", `available` / "Available", `degraded_but_serving` / "Degraded but serving", `awaiting_human` / "Awaiting human", `terminal_until_intervention` / "Terminal until intervention". Document how it derives from the **11 C6 technical lifecycle states** (`requested`, `preparing`, `ready`, `locked`, `changes_staged`, `dirty`, `committed`, `failed`, `inaccessible`, `unknown_provider_outcome`, `reconciliation_required`) per `DispositionLabelMapper.ResolveDisposition`, including the rule that **`ready` → `available` ONLY when there is no projection-lag evidence; with projection-lag evidence `ready` → `degraded_but_serving`**. State the F-4 rule that **disposition is primary and the technical state name is secondary muted metadata**, and that every status indicator is non-color-only (text + icon/shape + accessible label). Document the **seven no-mutation guarantees** (no mutation path, no credential reveal, no file-content browsing, no file editing, no raw-diff display, no hidden repair action, no unrestricted filesystem browsing) as the read-only / projection-backed / metadata-only console invariant.

3. Document the **perceived-wait UX and accessibility posture** in the console doc as **release-validation evidence, not CI gates**, without fabricating measured numbers. Perceived-wait (F-7): skeleton at **400 ms**, "still loading…" + Cancel at **2 s**, Cancel returns to the prior stable view (never an error state, never cancels a mutation — it only cancels an in-flight read). Release-validation page-load budgets (state these are **targets**, with measured numbers `reference_pending` and **no NBomber console harness / no console perf CI gate**): primary diagnostic page load **< 1.5 s p95 / < 3 s p99**, degraded/incident flows **≤ 5 s p95**, status/audit summary reads **< 500 ms p95**. Accessibility: **WCAG 2.2 AA**, single `<h1>` per page, `<html lang="en">`, `FocusOnNavigate` to the heading, captioned/scoped tables, English-only for MVP; automated structural conformance is verified by `AccessibilityContractSweepTests`, while manual checks (keyboard journeys, screen-reader, 125/150/200 % zoom, forced-colors, color-blindness) are `reference_pending` with the method defined. **Do not introduce a new accessibility or performance CI gate.**

5. Publish the **redaction-behavior documentation** (same `docs/operations/audit-and-redaction.md`, redaction section). Document the **visibly-distinct rule** (F-5 / architecture concern #11): **redacted MUST be visibly distinct from unknown and from missing; silent truncation or hiding is an operator-facing correctness bug**. Document the **two distinct vocabularies without conflating them**: the **wire** enum `RedactionVisibility` has **exactly 2 members** (`metadata_only`, `redacted`) plus a `reasonCode` (`^[a-z][a-z0-9_]{0,79}$`), while the **presentation** enum `FieldDisclosure` has **4 members** (`Visible`, `Redacted`, `Unknown`, `Missing`) with distinct `data-fc-disclosure` tokens — Redacted renders a lock-icon affordance, Unknown/Missing do not.
