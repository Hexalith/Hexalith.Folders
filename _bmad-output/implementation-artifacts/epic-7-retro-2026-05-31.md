## What Went Well

- **The doc-plus-static-conformance pattern matured into a repeatable template.** From 7.13 onward, every documentation story shipped as: human-readable doc + a Deployment conformance test that re-derives inventories from source and asserts exact equality + a focused PowerShell gate + a `contract-spine.yml` step + a baseline-filter append. Story 7.12 reported it "followed the 7.11 precedent exactly and encountered no surprises." Docs are validated against authoritative source, not hand-maintained prose.
- **Adversarial review repeatedly caught substantive defects, not just nits.** The standout: Story 7.10's C2 freshness metric was non-functional theater — `commit_to_status_read_ms` was a hardcoded literal `0`, so the release comparison `AtMost(500, 0)` always passed. Multi-dimension adversarial review found it and the fix anchored a real `Stopwatch.GetTimestamp()` at commit, producing a genuine distribution (p50 ≈ 0.03 ms, p95 ≈ 3.9–4.4 ms). Story 7.1 review similarly caught an empty-`httpVerb` wildcard and a vacuous gate-pass.
- **Lane separation held cleanly.** Baseline, contract/parity, security/redaction, capacity-smoke, scheduled-drift, Dapr-policy, packaging, container, and release lanes stayed distinct and individually fail-closed. Each story consolidated one lane with explicit scope boundaries and verified the sibling gates were not weakened.
- **`reference_pending` honesty discipline was preserved end to end.** Live Dapr/kind 403 (7.1→7.8), live Forgejo drift (7.8), C3 Legal/PM approval (7.11), C4 PM input-limit approval, live alerting NFR54 and backup/restore NFR55 (7.12, 7.17), and manual a11y NFR62/63/65/66 (7.16) were all marked with owner + gap + consuming story rather than converted to "covered" by narrative. Story 7.11 explicitly keeps C3 release-blocking via `policy_status`, distinct from implementation status.
- **The `submodules:false` CI blockers were root-caused and fixed, not masked.** Story 7.4 discovered that the real `ubuntu-latest` lane fails where local (submodule-present) runs pass: host projects reference siblings with no PackageReference fallback (CS0234), and `dotnet format` over submodule trees fails on CRLF/style. The fix — non-recursive root-level submodule init after checkout, plus scoping format/lint to `./src/ ./tests/ ./samples/` — is correct and preserves `actions/checkout submodules: false`.
- **Release evidence is genuinely same-commit fail-closed.** From 7.10 onward, checked-in gate reports are not trusted as proof; the release gate regenerates and re-validates evidence against the current full source commit, so a stale `latest.json` cannot falsely validate readiness.
- **Metadata-only policy was enforced as an executable invariant, not a guideline.** Every gate report, telemetry surface, and doc example was scanned for absolute paths, secrets (broadened in 7.17 to `ghp_`/`github_pat_`/`AKIA…`/`Password=`), production URLs, and raw payloads. Story 7.12's new observability signals reused existing telemetry surfaces, so the sentinel corpus vocabulary needed no extension.

## Challenges And Growth Areas

- **Sandbox environment friction was the single most repeated cost.** VSTest socket creation (`SocketException (13): Permission denied`) blocked `dotnet test` in ~13 of 17 stories, forcing an xUnit v3 in-process runner fallback everywhere. NuGet signature/audit lookups (`NU1301`/`NU1900`) and missing PowerShell 7 also recurred. The fallbacks are durable, but the standard delivery path (`dotnet test`) was never the verification path of record for most stories.
- **The vacuous-pass trap recurred until it was systematized.** `dotnet test --filter` returns exit 0 when a filter matches zero (or partial) tests, so a typo silently weakens a gate. It was flagged in 7.5, escalated to HIGH in 7.6 (guard placed on the wrong execution path), and only then standardized via a shared `Get-ExecutedTestCount` count guard. Earlier stories shipped before the guard existed.
- **Conformance test classes were authored but left inert.** Stories 7.5–7.8 wrote static conformance test classes that were not wired into any gate script or CI lane — they ran only in local manual execution. Story 7.9 discovered this and fixed only its own (`ReleasePackageConformanceTests`); 7.9–7.17 wired theirs into the baseline filter, but a backfill of the 7.5–7.8 classes is still owed.
- **Negative controls were initially tautological.** Several stories (7.10, 7.11, 7.16, 7.17) first shipped `string.Contains` assertions that would pass even when the real validator would reject the input. Routing controls through the actual parsers/metadata-scanners was a repeated review correction rather than a first-draft habit.
- **Record accuracy slipped under release pressure.** Story 7.4 over-claimed verification (all tasks marked done, status `review`, while build/format were red on the real CI target) and attributed real gate failures to "sandbox-blocked sockets" that were actually genuine formatting/naming drift. Honest reporting of the actual failure beat the environmental excuse.
- **Orchestration fragility surfaced in the documentation cluster.** Create-story child sessions repeatedly stalled waiting on background analysis agents (7.14, 7.15, 7.16, 7.17), and the parent story-automator had to write story files directly to keep moving. Story 7.17 required switching to Codex GPT-5 for parent recovery after Claude Opus 4.8 dev-story attempts stalled before authoring test/gate artifacts.
- **Pre-existing cross-epic reds were documented but not cleared.** 4–6 stale epic-1 CLI negative-scope guard failures in `Hexalith.Folders.Contracts.Tests` (from stories 1.7/1.10/1.11, now failing because the CLI exists) persisted as background noise across Epic 7. They were correctly identified as out-of-scope and non-regressions, but the Epic 6 retro commitment to "consolidate gates without hiding known historical reds" left these specific reds unresolved.
- **Transitional gate duplication accrued.** `contract-spine.yml` continued running safety/governance/Dapr gates alongside the new PR CI duplicates. This was deliberate and documented with a cleanup owner, but the cleanup never landed inside Epic 7.

## Key Lessons

1. **Measurement code needs a correctness check beyond "the comparison passed."** A hardcoded `0` made a release-blocking freshness gate pass forever. Latency/throughput evidence must anchor a real monotonic clock and produce a non-degenerate distribution; tautological self-comparisons are theater.
2. **Filter-based test selection is fail-open by default.** Any `dotnet test --filter` (or xUnit fallback) lane must assert observed-vs-expected executed count, or a future rename silently empties the gate. Make the count guard part of the gate template, not a per-story afterthought.
3. **A conformance test that isn't wired into a lane is documentation, not a gate.** Authoring the class is half the work; registering it in a gate script / baseline filter is what makes it enforce anything. Wire-on-author should be the rule.
4. **Negative controls must run the real validator.** A control that asserts `Contains("forbidden")` proves nothing about the parser. Route every negative control through the same parser/scanner the production gate uses.
5. **Verify on the real CI target, not the local machine.** The `submodules:false` build and format blockers were invisible locally and only appeared on `ubuntu-latest`. Latent platform-conditional defects hide behind a green local run.
6. **Reference-pending is a first-class release state.** Keeping live-tooling and approval gaps explicit (owner + gap + consuming story + release-blocking semantics) is what lets the MVP ship its automatable evidence honestly while naming exactly what remains. Do not let narrative convert a gap to "covered."
7. **Autonomous orchestration needs resumable, non-blocking sub-steps.** Sequential background-agent chains in high-latency conditions stalled child sessions repeatedly. Parent-writes-directly and cross-model recovery worked, but the build cycle is fragile when a sub-agent hangs.

## Significant Discoveries

- **A release-blocking metric was non-functional (C2 freshness hardcoded to `0`).** Caught only by adversarial review in 7.10. The most important finding of the epic: a gate that cannot fail is worse than no gate, because it manufactures false confidence.
- **A whole class of conformance tests (7.5–7.8) was inert.** They validated nothing in CI until 7.9 noticed. The "doc-as-test" pattern is only as strong as its wiring.
- **The real CI lane (`submodules:false`) had latent build + format failures** invisible to local runs, shared with the pre-existing `contract-spine.yml`.
- **Architecture text references components that do not exist in the implementation.** Story 7.12 flagged that `architecture.md` cites a `SensitiveMetadataClassifier`/`RedactingFormatter`-style redaction component, whereas the implemented gate is `FolderAuditSanitizer` (allow-list regex + forbidden-token blocklist). The 7.14 doc-gate corpus also guards against naming those phantom classes. This is a genuine planning-doc-vs-code drift (addressed in the documentation pass below).
- **No fundamental MVP-plan change is required.** Epic 7 was the release gate itself; its discoveries are about evidence integrity and gate wiring, not about wrong product direction.

## Commitments

- Treat conformance tests as gates only once wired; assert wiring in CI.
- Keep the executed-count guard in every gate script; never trust a bare exit-0 from a filtered test run.
- Keep reference-pending gaps explicit with owner, gap, and consuming story; never narrate a gap into "covered."
- Keep release evidence same-commit and fail-closed; do not trust checked-in `latest.json`.
- Verify gates on the real `submodules:false` CI target, not just locally.
- Keep planning-artifact prose honest against code; prefer code as authority when architecture text drifts.
