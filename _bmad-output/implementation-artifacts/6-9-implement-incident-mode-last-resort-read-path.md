# Story 6.9: Implement incident-mode last-resort read path

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As an operator,
I want an ACL-checked incident stream when projections are degraded,
so that diagnosis can continue while read models recover.

## Acceptance Criteria

> **Epic 6.9 BDD (verbatim, epics.md L1429–1434):**
> **Given** projections are degraded and the actor has incident permission
> **When** `/_admin/incident-stream` renders
> **Then** it shows a persistent degraded-mode banner, raw event metadata, disposition labels from Story 6.3, and correlation/time-window copy affordance
> **And** redacted values render through the shared redaction component from Story 6.4 with no relaxed policy.

> **CRITICAL SCOPING FACT (read before coding):** There is **no** dedicated raw-event-stream / incident endpoint in the OpenAPI Contract Spine or the generated SDK, and the UI is forbidden from direct EventStore access (F-2, concern #11). The only metadata-only, time-ordered, "for incident reconstruction" read available to the console is the **already-shipped** folder-scoped `IClient.ListOperationTimelineAsync(...)` (Story 6.1). This story therefore delivers the F-6 **operator experience** (persistent degraded banner + disposition-beside-raw-event + redaction-with-no-relaxation + correlation/time-window copy + server-enforced ACL) **on top of the existing operation-timeline read** — it does **not** add or change any endpoint, SDK method, generated client, package, or OpenAPI content. The deeper architectural vision of a projection-independent authoritative event-stream read is **reference-pending** (a Workstream-7 / release concern), not shippable in this MVP without touching frozen contracts. See Dev Notes → "Scope & the projection-independence variance".

Decomposed, testable acceptance criteria:

1. **Incident-stream page** exists at `@page "/_admin/incident-stream"`. Page root `<div data-testid="console-page-incident-stream-root">`, exactly one `<h1>` (e.g. `Incident stream · @Folder`). Folder context is supplied via `[Parameter, SupplyParameterFromQuery] string? Folder` and pagination via `[Parameter, SupplyParameterFromQuery] string? Cursor`. (Concrete route fixed here per wireflow §1.2 note "each owning story fixes its concrete route"; folder is a query param, not a path segment, so the literal F-6 route `/_admin/incident-stream` is preserved.)
2. **Persistent degraded-mode banner (F-6 guardrail 1)** renders a **new `DegradedModeBanner` component** that is **always present** while on this page (unconditional — this *is* the degraded-mode page; the banner is not gated on detecting degradation). It shows the text *"DEGRADED MODE — events shown may be incomplete or out of order."* plus *"Last projection checkpoint: `<HH:MM:SS UTC>`"*, conveyed by **text + icon + colour, never colour alone** (UX-DR14). `data-testid="incident-degraded-mode-banner"`; accessible as a labelled region/`role="status"`. The checkpoint timestamp is the **freshness `ObservedAt`** of the operation-timeline read, formatted UTC; an absent/`default` observed-at renders **"unknown"**, never a fabricated `0001-01-01` and never "Current" (UX-DR26 freshness honesty).
3. **Raw event metadata (F-6 guardrail 2 / "raw event metadata"):** when a folder is supplied and authorized, render the `OperationTimelinePage.Entries` newest-first as the incident event list. Each row surfaces, where authorized: evidence **timestamp**, **operation** id, **task** id, **correlation** id, **workspace reference**, the **state transition** `from → to` (the raw technical lifecycle vocabulary via `<TechnicalStateMetadata>`), **sanitized result**, **retryable** posture (advisory, never an action), and **duration** (ms).
4. **Disposition labels from Story 6.3 rendered alongside the raw event type (F-6 guardrail 2):** every event row renders `<OperatorDispositionBadge Disposition="@row.Disposition" />` **beside** the raw technical state transition so operators do not switch vocabularies mid-incident. The disposition is the **server-computed `StateTransition.Disposition` passed through verbatim** (never re-derived from `ToState`).
5. **Correlation + time-window copy affordance (F-6 guardrail 3):** a **new `CorrelationCopyButton` component** provides a one-click, **read-only** clipboard copy (JS `navigator.clipboard.writeText`, `<button type="button">`, no `form`/dialog — must not trip the mutation guard) of a **metadata-only** handoff payload: the page read **correlation id** + the **timestamp window** of the currently-shown events (oldest..newest `EvidenceTimestamp`, UTC). `data-testid="incident-correlation-copy"`; keyboard reachable with an accessible label. The payload contains only correlation id + UTC timestamps — never secrets, paths, file contents, or provider payloads.
6. **Redaction applied with NO relaxed policy (AC line 2 / F-5 / §3.6):** every redactable field routes through the **shared shipped `<RedactedField>`** fed by `RedactionDisclosureMapper`, using the **identical** mapping the operation-timeline page uses — redaction does **not** relax just because projections are degraded. Reuse `OperationTimelineEntryView.Create(entry)` to assemble each row (do **not** reinvent the mapping): workspace reference is redactable (`FromAuditRedaction`); ids (`OperationId`/`TaskId`/`CorrelationId`/`TimelineEntryId`) render via `<SafeCopyId>` and `<RedactedField Disclosure="Missing">` when blank; the evidence timestamp is a plain `DateTimeOffset`. `Redacted` ≠ `Unknown` ≠ `Missing` ≠ `Visible` stay visibly/semantically distinct; only `Redacted` shows the lock icon; **a redacted value is never emitted to the DOM**.
7. **Tenant scope rendered (scope-before-evidence, UX-DR4/DR6):** `<TenantScopeBanner Permissions="@_permissions" />` renders above the `<h1>`, sourced from `IUserContextAccessor.TenantId` **internally** — **never** from the `?folder=` query, `?cursor=`, or any client input. The supplementary permissions read uses the swallow-denial `TryReadAsync` helper.
8. **Incident permission is enforced server-side (safe denial, UX-DR21):** the page does **not** implement a client-side authorization gate on a self-asserted claim. The ACL/`eventstore:permission=admin` decision is the server's: a `HexalithFoldersApiException` on the `ListOperationTimelineAsync` read renders `<ConsoleErrorPanel>` (via `ConsoleErrorPresenter.FromException`), showing the **canonical category token only** (never raw server text, never a `not_found`-vs-`*_denied` existence oracle), and **suppresses the event table**. The degraded banner + scope banner + single `<h1>` still render. The `Administer` posture (`FolderPermissionLevel.Administer`) may be surfaced advisorily via the scope banner, but is **not** the authoritative gate.
9. **Folderless / empty / unavailable states (UX-DR20):** the page root + single `<h1>` + degraded banner render in **every** state. (a) No `?folder=` supplied → an instruction/empty state guiding the operator to open the stream for a specific folder (`<ConsoleEmptyState>` with an appropriate reason or a labelled instruction paragraph) — never an error, never a fabricated event list. (b) Zero authorized rows → `<ConsoleEmptyState Reason="EmptyStateReason.NoMatches" />`. (c) Transport failure (`HttpRequestException`/`TaskCanceledException`) or projection-down → `<ConsoleEmptyState Reason="EmptyStateReason.ReadModelUnavailable" />`.
10. **Pagination is cursor-based and read-only:** the `Cursor` query parameter drives the load; a "Next page" **`<a href>`** (never a button/form) appears when `Page.IsTruncated` with a non-blank `Page.Cursor`, otherwise an "End of results" marker. The next-page href preserves the `?folder=` value. No client-side filtering/hiding of returned rows.
11. **"Filtered" = server capability only — NO filter UI (C4):** pass `filter: null` on every `ListOperationTimelineAsync` call; render no filter control implying unsupported filtering works.
12. **Read-only / metadata-only (F-2 / concern #11):** the page passes `ShouldHaveNoMutationAffordances()` (no `form`/`fluentinputform`/`fluentdialog`/`[data-fc-command]`/`[data-fc-mutation]`), registers no command manifest, and renders no file contents, raw diffs, raw/absolute paths, secrets, credential values, or unauthorized-resource existence. A pure read-render page consuming an existing DTO introduces **no new safety channel** (no `safety-channel-inventory.json` change).
13. **Connected evidence (UX-DR19):** the incident stream is reachable, not orphaned — add a cross-link to `/_admin/incident-stream?folder={FolderId}` from `FolderDetail` (and/or the `OperationTimeline` page), and have the incident page cross-link back to the folder's operation-timeline / audit-trail. The incident-stream link copy frames it as the **last-resort** path.
14. **Out of scope — do not pull neighbouring stories in (review-failure flags):** **F-7 perceived-wait UX (skeleton at 400 ms, "still loading…[cancel]" at 2 s) is Story 6.10** — render a simple labelled `data-testid="console-page-incident-stream-loading" aria-busy="true"` paragraph only; do not build `SkeletonState`/`StillLoadingCancel`. **Formal WCAG/no-mutation release verification is Story 6.11** — build accessibly (one `<h1>`, non-colour-only status, keyboard-reachable links/copy), but the formal audit is 6.11. **No new endpoint, SDK method, generated client, package, or OpenAPI edit.**
15. **Tests + gates:** bUnit page tests (`tests/Hexalith.Folders.UI.Tests`) cover render in all states (folderless / authorized / safe-denial / empty / unavailable), the persistent banner, raw-event field set, disposition-beside-raw, redaction distinctness + no-leak, the copy affordance (no-mutation + composed payload), freshness honesty, pagination, and `ShouldHaveNoMutationAffordances()`; component tests cover `DegradedModeBanner` (always-present, checkpoint honesty, non-colour-only) and `CorrelationCopyButton` (no-mutation, accessible, copies the composed value). An E2E route smoke test is added with a `ConsoleRoutes.IncidentStream(folderId)` builder. The full UI lane stays green over the **422/422** post-6.8 baseline, with **zero** edits to `src/Hexalith.Folders.Client/Generated/`, `Directory.Packages.props`, the OpenAPI spine, or any `*.csproj`/`*.slnx`.

## Dev Notes

**Scope is UI-only, and built on an existing read.** This story adds Blazor components/pages in `Hexalith.Folders.UI` that consume the **already-shipped** typed SDK (`IClient.ListOperationTimelineAsync` from Story 6.1, frozen by the Contract Spine / NSwag generation in Stories 1.11/1.12) and reuse the shipped console components. It does **not** add or change any endpoint, SDK method, generated client, package, or OpenAPI content.

### Scope & the projection-independence variance (read first)

- The architecture's F-6 vision (architecture.md L550) describes a *"separate ACL-checked event-stream view … available when projections are degraded … surfaces latest events … for operators with `eventstore:permission=admin`"* — implying a read path to **authoritative event streams** that survives when the projections themselves are broken.
- **That projection-independent read is not shippable in this MVP** without violating frozen constraints: (a) the OpenAPI spine + generated client are forbidden-touch and expose **no** raw-event/incident endpoint (verified: `grep` of the spine + `src/Hexalith.Folders.Client/Generated/` finds only the audit/timeline "incident reconstruction" methods); (b) the UI is forbidden direct EventStore access (F-2, concern #11 — "reads only from projections via the SDK"); (c) every SDK read method is folder-scoped.
- **As-built decision (honest and shippable):** deliver the F-6 **operator experience and guardrails** on top of the existing folder-scoped `ListOperationTimelineAsync` (the explicit "metadata-only … for incident reconstruction" surface, the closest available "raw event metadata"). Treat each operation-timeline entry as a raw event row (state transition = the raw technical event vocabulary). Surface the deeper projection-independence as **reference-pending** (a release / Workstream-7 concern), exactly as the console already treats C2/C3/C4/C5 reference-pending markers (wireflow §7). **Document this variance in the Dev Agent Record** so the reviewer does not flag a missing endpoint as a defect.
- This satisfies AC line 1 ("raw event metadata, disposition labels …, correlation/time-window copy affordance") and the wireflow §3.5 component plan (`DegradedModeBanner`, `IncidentStream`, `CorrelationCopyButton`) without fabricating contract surface.

### Technical requirements (dev-agent guardrails)

- **Bind to the SDK (NSwag) types, NOT server records.** `Hexalith.Folders.UI` references `Hexalith.Folders.Client` only. Use `Hexalith.Folders.Client.Generated.*`: `OperationTimelinePage` (`.Entries`, `.Page` (`PaginationMetadata`: `IsTruncated`/`Cursor`/`TruncatedReason`), `.Freshness` (`FreshnessMetadata`: `Stale`/`ObservedAt`), `.RetentionClass`), `OperationTimelineEntry` (`TimelineEntryId`, `EvidenceTimestamp` plain `DateTimeOffset`, `OperationId`/`TaskId`/`CorrelationId` plain strings, `WorkspaceReference` `RedactableDiagnosticIdentifier`, `StateTransition` `DiagnosticStateTransition` (`FromState`/`ToState` `LifecycleState` enums, `Disposition` `OperatorDispositionLabel` enum), `SanitizedResult` `CanonicalErrorCategory` enum, `Retryable` bool, `DurationMilliseconds` int). No `Enum.Parse`/string-parsing — the generated client deserializes wire strings into enums. (A parallel server-side `Hexalith.Folders.Contracts.Projections.*` record set with `string` fields exists — **ignore it; the UI never references Contracts.**)
- **SDK method (interface `IClient`):** `Task<OperationTimelinePage> ListOperationTimelineAsync(string folderId, string x_Correlation_Id, ReadConsistencyClass? x_Hexalith_Freshness, string cursor, int? limit, string filter)`. Call the **no-`CancellationToken`** overload (as all existing pages do). `GetOperationTimelineEntryAsync(...)` exists for an optional detail route — **not required** by this story.
- **Call discipline (mirror `OperationTimeline.razor.cs` exactly — it is the template):**
  - One fresh correlation id per page load: `_correlationId = Guid.NewGuid().ToString()`, passed to every call, surfaced via `CorrelationCopyButton` + `SafeCopyId`.
  - `ReadConsistencyClass.Eventually_consistent` on every read (eventually-consistent browsing — never the active snapshot probe). Even in "incident" framing, this is still a passive read; do not invent a different consistency mode.
  - `filter: null` on every call (C4 rejection-only — a non-null filter returns `400`/`filter_not_yet_supported`). `limit: PageLimit` (const 50; server clamps).
  - `.ConfigureAwait(false)` on **every** await (CA2007 is warnings-as-error in the UI project).
  - Override `OnParametersSetAsync` (not `OnInitializedAsync`) — query params (`Folder`, `Cursor`) drive the load and it re-runs on change.
  - Primary read `try/catch`: `HexalithFoldersApiException` → `ConsoleErrorPresenter.FromException(ex, _correlationId)` → `<ConsoleErrorPanel>`; `HttpRequestException`/`TaskCanceledException` → `_unavailable = true` → `<ConsoleEmptyState Reason="EmptyStateReason.ReadModelUnavailable" />`. Supplementary permissions read via the private static `TryReadAsync<T>(Func<Task<T>>)` that swallows those → `null`.
- **Persistent banner ≠ conditional banner.** The `DegradedModeBanner` is rendered **unconditionally** on this page (it *is* the degraded-mode surface reached via a runbook link), in every branch including loading/denial/empty/unavailable. Do not gate it on detecting staleness — that would defeat the F-6 intent ("sleep-deprived operator following a runbook link must not land on a raw event stream with zero scaffolding").
- **Checkpoint timestamp = freshness honesty.** "Last projection checkpoint" is the timeline read's `Freshness.ObservedAt` (UTC, `"u"` format); absent/`default` → "unknown". Reuse the `OperationTimeline` `ObservedAtLabel()` discipline verbatim — never present `0001-01-01` or "Current" for absent freshness (UX-DR26; the recurring 6.7/6.8 freshness-honesty review fix).
- **Redaction reuse, no relaxation.** Assemble rows with `OperationTimelineEntryView.Create(entry)` (already maps `WorkspaceReference` via `RedactionDisclosureMapper.FromAuditRedaction` and nulls non-Visible values). Render exactly as `OperationTimeline.razor` does. The §3.6 contract is explicit: incident mode does **not** relax redaction. Tests must assert a redacted workspace value is absent from `Markup`.
- **Disposition primary/alongside (F-4 + guardrail 2):** pass the server-computed `entry.StateTransition.Disposition` straight to `<OperatorDispositionBadge>` (via the view-model's `Disposition`); render it **beside** the raw `<TechnicalStateMetadata>` from→to. Do **not** re-run `DispositionLabelMapper.ResolveDisposition` over `ToState` (the 6.7/6.8 "prefer the server value" lesson).
- **Copy payload is metadata-only.** `CorrelationCopyButton` copies only `correlationId` + a UTC timestamp window. Never include secrets, tokens, raw paths, file contents, provider payloads, or unauthorized-resource existence. The window is derived from the visible entries' `EvidenceTimestamp` (skip `default`).
- **Tenant authority** is server-side from authenticated claims + EventStore envelope; the UI sends no tenant id and reads `?folder=` only as a comparison input passed to the SDK (the server authorizes it). `TenantScopeBanner` reads `IUserContextAccessor.TenantId` for display only.

### Architecture compliance

- **F-1/F-2/F-3:** Blazor Server (Interactive Server, set once globally — **no per-page `@rendermode`**), reads only from projections via the `Hexalith.Folders.Client` SDK, Fluent UI via the FrontComposer/Shell pattern. No direct EventStore aggregate access; no `AddHexalithEventStore`.
- **F-6 (this story):** the three guardrails are mandatory and load-bearing — (1) persistent red degraded banner with last-checkpoint, (2) operator-disposition labels beside raw event types, (3) one-click copy correlationId + time window. **F-5 lock icons still apply** — redaction does not relax (architecture L550, wireflow §3.5/§3.6).
- **Out of scope — do not pull neighbours in:** F-7 perceived-wait (skeleton/cancel) is **6.10** (simple `aria-busy` loading paragraph here); formal WCAG/no-mutation audit is **6.11**; no filter UI (**C4**).
