# ADR 0002: Contract Spine as the single source of truth

Date: 2026-05-31

Decision identifiers: `C0`, `A-1`, `A-2`, `A-3`, `C13`. Implementing epic: Epic 1 (Contract Spine). This is a retrospective ADR; it records a decision already implemented across Epics 1-7, it does not propose new design.

## Status

Accepted. The Contract Spine has been the governing artifact since the Epic 1 Contract Spine stories and is enforced by the `contract-spine-gates` workflow today.

## Context

Hexalith.Folders exposes the same folder lifecycle behavior over four surfaces: REST, the typed SDK, the CLI, and the MCP server. Without one authoritative contract, each surface drifts: error codes diverge, idempotency canonicalization is re-implemented per client, and parity claims become aspirational rather than tested.

Architecture decision `C0` names a single source-of-truth artifact for cross-surface generation, and `A-1` pins that artifact to OpenAPI 3.1 owned by `Hexalith.Folders.Contracts`. The contract carries an extension vocabulary for idempotency keys, correlation, lifecycle states, parity-dimension annotations, and audit-metadata declarations.

## Decision

The OpenAPI 3.1 Contract Spine at `src/Hexalith.Folders.Contracts/openapi/hexalith.folders.v1.yaml` is the single source of truth for every consumer surface.

- `A-2`: the typed SDK in `Hexalith.Folders.Client` is generated by NSwag, and NSwag templates emit a per-command `ComputeIdempotencyHash()` helper so consumers never re-author canonicalization.
- `A-3`: REST is a parallel transport in `Hexalith.Folders.Server`; the server-generated OpenAPI is validated against the spine in CI as a blocking gate, not an advisory check.
- `C13`: the parity oracle `tests/fixtures/parity-contract.yaml` is derived from the spine and declares the transport-parity and behavioral-parity columns every surface test consumes.

Generated client files under `src/Hexalith.Folders.Client/Generated` are never hand-edited; the spine or the generation pipeline changes and the artifacts are regenerated.

## Consequences

- Surfaces stay synchronized by construction: a contract change forces regeneration of the SDK, server OpenAPI, parity oracle rows, and idempotency helpers together.
- The blocking CI gate (`tests/tools/run-contract-spine-gates.ps1`) rejects server-vs-spine and generated-client drift before merge, so parity is tested rather than asserted.
- The cost is process discipline: any error-shape, result-code, action-token, or parity-dimension change is a public contract change that must update OpenAPI, fixtures, docs, tests, and generated outputs in one move.

## Alternatives Considered

- Hand-written per-surface contracts were rejected because they make parity unverifiable and let canonicalization drift across clients.
- Letting the server be the source of truth (and generating the contract from server reflection) was rejected because it couples the contract to one transport and weakens the SDK/CLI/MCP parity guarantees that `C13` exists to test.
