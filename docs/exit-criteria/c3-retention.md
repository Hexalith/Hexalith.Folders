## Rationale

C3 is Phase-1-blocking because the Contract Spine must expose D-7 idempotency TTL semantics before client and parity work starts. The proposed durations keep audit and reconciliation evidence long enough to support operational review while keeping temporary working files short-lived because they are disposable caches, not authoritative state.

No row invents a final legal policy. The proposed values are concrete enough for review, and every row names the authority gap that must close before Phase 1 uses the values as binding defaults.

## Verification impact

Verification must prove this artifact exists, contains no generic placeholders, contains only metadata-only examples, and maps each duration to a consuming future story or artifact. Later retention implementation must add tests for tenant-prefixed keys, deletion/tombstone behavior, audit replay, and commit idempotency TTL inheritance.

## Deferred implementation

This document does not add cleanup workers, projection compaction, tenant-deletion flows, audit endpoints, Dapr state configuration, OpenAPI fields, or idempotency code. Story 1.6 must not treat C3 as unblocked until Legal + PM approve or revise the proposed values.
